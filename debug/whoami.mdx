## How to read this

- **`groups`** is what you filter pages on. Add `groups: ["..."]` to a page's frontmatter (or a nav group in `docs.json`) — a user must be in at least one listed group to view it; others get a 404.
- **`content`** is your personalization payload — reference it in any page with `{user.content.fieldName}`.
- Mintlify fails closed: if the configured claim is missing from the auth response, sign-in is blocked entirely. So a successful login with an empty `groups` array means your backend returned `groups: []`, not that Mintlify dropped it.
