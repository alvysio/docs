If the raw claims below show `no user in scope`, the auth handshake did not return a user object. For JWT, confirm the `host` claim exactly matches your docs domain and that your token/info endpoint includes `groups`.

## Quick checks

- **Groups:** {user?.groups?.length ? user.groups.join(", ") : "none"}
- **Session expires at:** {user?.expiresAt ? new Date(user.expiresAt * 1000).toISOString() : "none"}
- **Personalization content keys:** {user?.content ? (Object.keys(user.content).join(", ") || "empty object") : "none"}
- **API playground input keys:** {user?.apiPlaygroundInputs ? (Object.keys(user.apiPlaygroundInputs).join(", ") || "empty object") : "none"}

## Raw claims received

<pre>{JSON.stringify(user ?? { note: "no user in scope: you are not authenticated" }, null, 2)}</pre>

## How to read this

- **`groups`** is what you filter pages on. Add `groups: ["..."]` to a page's frontmatter (or a nav group in `docs.json`); a user must be in at least one listed group to view it, and others get a 404.
- **`content`** is your personalization payload. Reference it in any page with `{user.content.fieldName}`.
- Mintlify fails closed: if the configured claim is missing from the auth response, sign-in is blocked entirely. A successful login with an empty `groups` array means your backend returned `groups: []`, not that Mintlify dropped it.
